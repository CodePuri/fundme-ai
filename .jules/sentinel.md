## 2025-02-20 - Fix missing input validation and information leakage in Onboarding API
**Vulnerability:** The `/api/onboarding` API lacked input length and protocol validation for user-submitted fields, posing a DoS and injection risk. Furthermore, Supabase database errors (`error.message`) were directly returned to the client, leading to internal information leakage.
**Learning:** In restrictive environments without libraries like `zod`, manual input validation is strictly required. Relying on database-level limits is insufficient because oversized payloads can consume application resources. Also, error objects from backend systems (like Supabase) often contain stack traces or schema details.
**Prevention:** Implement custom validation logic for all API endpoints enforcing length limits and exact schemas (e.g., URL protocols). Always log detailed errors securely server-side using `console.error` and return only generic messages like "Internal server error" to the client.
