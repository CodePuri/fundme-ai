## 2025-05-02 - Next.js Middleware Misconfiguration
**Vulnerability:** The Next.js middleware file was incorrectly named `proxy.ts` instead of `middleware.ts`. This misconfiguration caused Next.js to ignore the middleware entirely, resulting in silent failures and bypassing all authentication and route protections (CRITICAL authentication bypass).
**Learning:** Next.js relies on strict file naming conventions for core features like middleware. If the file is not named `middleware.ts` or `middleware.js` and placed in the root (or `src/` directory), it simply won't execute, and the framework will not provide any explicit warning or error about the missing protections.
**Prevention:** Always verify that critical framework-specific files (like `middleware.ts`) strictly adhere to the framework's naming conventions. Additionally, implement automated tests that specifically verify that protected routes successfully reject unauthorized access, rather than just testing that the middleware logic works in isolation.
